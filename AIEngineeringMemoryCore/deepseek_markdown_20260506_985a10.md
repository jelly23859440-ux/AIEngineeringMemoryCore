# AIEngineeringMemoryCore (AI记忆中枢)

> 如何正确使用AI的能力，搭建可靠的运行架构，才是接下来AI影响现实世界的基石。

---

## 一、概述

### 为什么做这个
大语言模型从来不是能力不够。它们的瓶颈在于**架构缺位**。

让一个最强模型处理持续数周的项目，本质上等于让一个记忆力只有几秒的天才反复从头开始。窗口无限扩大？成本骤增、注意力衰减——这不叫解决，叫硬扛。手工管理记忆？碎片化、不可靠、还跟模型绑死。

**根本症结**：我们一直在要求一个没有记忆系统的模型去完成需要记忆的任务。正确的做法不是在模型里塞更多东西，而是在模型外面，为它搭建一个记忆系统。

### AIEngineeringMemoryCore 是什么
一个**本地运行、模型无关**的记忆引擎。它模拟人脑的记忆巩固机制：工作时监听推理节奏，在上下文将满前自动触发“心跳巩固”，将对话历史压缩为**情景骨架 + 精确实体索引**双层记忆；下一次AI重新进入任务时，自动激活相关记忆注入上下文，让AI像人一样“记得之前做到哪了”。

它不绑定任何模型。记忆文件存储在你的本地。一切由你掌控。

### 核心理念
1. **承认模型的边界，用架构补足**——模型有物理上限，记忆中枢在边界处架起延伸的桥梁。
2. **记忆是结构，不是日志**——压缩归纳，情景骨架快速重建认知，实体索引高保真回溯。
3. **用户永远掌握控制权**——记忆存本地，巩固用本地模型，全部可配置。
4. **可靠的架构，而不是更强的模型，才是基石**——窗口限制和遗忘规律不会消失，稳定的记忆架构比等待下一个更强模型更值得投入。

## 二、操作说明

### 它如何工作（一句话版）
把你的项目文件夹放进AI记忆中枢，它在后台自动帮你压缩历史、记录细节。下次你继续项目时，AI会记得之前做到哪了。

### 第一步：安装与准备
1. 将 `AIEngineeringMemoryCore/` 整个文件夹放入你的项目根目录。
2. 确保本地已安装并运行一个可用的记忆管理模型（推荐使用 Ollama，默认模型 `llama3`）。如果倾向于使用云端API，也可配置为任意OpenAI兼容接口。
3. 依赖安装（首次使用时）：
   ```bash
   cd AIEngineeringMemoryCore
   pip install -r requirements.txt
第二步：首次启动
用 Trae 打开你的项目。

Trae 会自动扫描项目根目录，发现 AIEngineeringMemoryCore/ 文件夹后，在界面提示：

“检测到项目包含 AI记忆中枢。是否启动后台记忆管理？[启动] [跳过]”

选择启动后，Trae 引导你完成一次性配置：

“请提供记忆管理AI的API信息：”

API端点（例如 http://localhost:11434/v1）

API Key（如不需要可留空）

模型名称（默认 llama3）

记忆存储位置（默认 当前项目/.memory/）

配置完成后，AI记忆中枢在后台启动，终端输出：

“AI记忆中枢已后台启动，正在管理本项目记忆。”

后续再次打开项目时，会自动读取已有配置，无需重复设置。

第三步：日常使用（完全无感）
配置完成后，你不需要做任何额外操作。在 Trae 界面正常对话即可。每次对话背后，AI记忆中枢自动完成两件事：

发送消息前：检索相关记忆，静默注入上下文。你的执行AI会收到类似这样的前置信息：

text
[系统记忆回溯 - AI记忆中枢]
任务摘要：正在开发电商系统，已完成用户模块，当前在支付模块。
关键决策：使用 Stripe 作为支付网关。
待解决问题：退款回调的timeout值未定。
相关精确实体：
- payment_gateway = "stripe" [可信度: high]
- refund_callback_timeout = 5000ms [可信度: medium，曾报错超时]
收到回复后：自动记录本轮对话，累计 token 数。当 token 接近模型上限时，后台静默触发巩固，调用记忆管理AI生成新的记忆包，存入本地 .memory/ 文件夹。

第四步：子任务完成时（按需操作）
当你一个子任务告一段落时，可以告诉 Trae：“这个子任务完成了”。AI记忆中枢会提示：

“请为此子任务设置几个唤醒关键词（用逗号分隔）：”

你输入关键词后（例如 支付模块,Stripe集成,退款回调），该子任务的所有记忆转为“冷数据”归档。以后只有当你的提问命中这些关键词时，相关记忆才会被唤醒，日常不消耗任何算力。

记忆协作协议
执行AI启动时，系统提示词中会自动追加以下协议，确保执行AI知道如何使用记忆中枢提供的索引：

优先采用标记为 [可信度: high] 的记忆

对 [可信度: medium] 的记忆可参考但自行判断

对 [可信度: low] 或 [manual_corrected] 的记忆谨慎参考

若发现记忆与事实矛盾，明确指出并建议用户更新

需要精确值时，直接引用并说明来源

记忆存储位置
所有记忆文件存储在 <你的项目>/.memory/ 目录下：

summaries/ —— 情景骨架摘要

entities/ —— 精确实体索引

cold/ —— 冷记忆归档

heartbeat_state.json —— 心跳计数状态

你可随时查看、备份或删除这些文件。删掉 .memory/ 文件夹等于清空所有记忆。

配置修改
如需修改记忆管理AI的配置，编辑 AIEngineeringMemoryCore/config.yaml：

yaml
management_ai:
  api_base: "http://localhost:11434/v1"
  api_key: ""
  model: "llama3"

memory:
  storage_path: "./.memory"
  heartbeat_ratio: 0.8
  cold_storage_enabled: true

memory_manager:
  mode: "hybrid"
  async_heartbeat_batch: 60
修改后重启 Trae 即可生效。

三、开发任务清单（给 Trae 的开发流程）
总览
阶段	任务	说明

第3步	心跳监控器	token累计，80%阈值触发巩固，状态持久化
第4步	巩固模型适配器	调用管理AI生成MemoryPacket，支持本地和云端
第5步	冷热分级管理	子任务完成标记，触发词绑定，冷记忆休眠
第6步	记忆检索器	固定条数返回，本地嵌入，遵守冷热规则
第7步	记忆管理AI调度器	三种模式，异步/实时/混合，紧急呼叫接口
第8步	记忆整理处理器	去重、纠错、索引重构，可信度回写

各任务详细说明
第1步：记忆数据结构
定义 models.py 中的核心数据类：

EpisodeSummary：摘要文本、时间戳、可信度、关联实体ID列表、冷热标记

EntityRecord：实体名、类型、精确值、可信度等级（high/medium/low/manual_corrected）

MemoryPacket：一次巩固产出的记忆包，包含摘要和实体列表

实现与本地文件系统的序列化与反序列化

第2步：项目隔离
实现 ProjectManager：

传入根目录路径，自动识别当前项目

在项目目录下自动创建 .memory/ 子目录结构

不同项目间的记忆严格隔离

第3步：心跳监控器
实现 HeartbeatMonitor：

提供 track_message(msg) 方法供外部调用

内部使用 tiktoken 估算累计 token 数

当累计 token 达到模型上下文上限的 80% 时触发巩固回调

心跳计数持久化到 .memory/heartbeat_state.json

使用文件锁保证单进程安全，注释标注多进程扩展方向

第4步：巩固模型适配器
实现 Consolidator：

支持独立配置管理AI接口，与执行AI分离

默认连接本地 Ollama，也可配置为任意OpenAI兼容API

调用管理AI将对话历史转化为结构化 MemoryPacket

错误重试、JSON解析失败降级为纯文本摘要

提示词模板要求移除密钥、令牌等敏感信息

第5步：冷热分级管理
实现 ColdStorageManager：

提供 mark_subtask_complete(task_id, keywords) 方法

将指定子任务记忆标记为冷记忆，绑定触发词

检索时默认只查热记忆，命中触发词时才唤醒冷记忆

通过API暴露归档端点，或在SDK中直接调用

第6步：记忆检索器
实现 MemoryRetriever：

固定条数返回，可配置 Top-K 摘要和 Top-M 实体

使用本地 Chroma 做向量检索，嵌入模型默认使用 Ollama 的 nomic-embed-text

返回结果附带可信度标记

严格遵循冷热分级规则

第7步：记忆管理AI调度器
实现 MemoryManagerScheduler：

支持三种模式：async（心跳累计满60次后批量整理）、realtime（每次检索前整理）、hybrid（异步为主+紧急呼叫接口）

所有参数可在 config.yaml 中配置

异步整理不阻塞主流程

第8步：记忆整理处理器
实现 MemoryConsolidationProcessor：

接收一批待整理记忆，调用管理AI执行去重、纠错、索引重构

输出整理后的 MemoryPacket，备份被淘汰的旧条目

将可信度评分回写到实体记录

第9步：Trae 集成启动器
实现 start.py：

作为 AIEngineeringMemoryCore 在 Trae 项目中的启动入口

检测是否存在 config.yaml，若不存在则引导用户完成首次配置

若已配置则静默初始化，启动 AIMemoryHub 实例

在执行AI的系统提示词中自动追加“记忆协作协议”

终端输出启动成功提示

开发顺序
严格按照第1步至第9步顺序推进，每一步依赖前一步提供的数据结构或模块。

所有功能完成后，补充：

examples/ 目录下的示例用例

README.md 中的快速开始章节

各模块的单元测试
